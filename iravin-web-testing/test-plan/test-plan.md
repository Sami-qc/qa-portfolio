# Test Plan — IRAVIN Website

## 1. Objective
To verify that the IRAVIN e-commerce website works correctly across key user flows and identify functional and usability issues.

## 2. Scope

### In Scope
- Homepage and banners
- Navigation menu
- Search functionality
- Product listing pages
- Product detail pages
- Cart behavior
- Checkout entry flow
- Language and currency switching
- Responsive behavior

### Out of Scope
- Backend/API testing
- Payment processing validation
- Performance testing
- Automation testing

## 3. Testing Type
- Functional Testing
- UI/UX Testing
- Negative Testing
- Exploratory Testing
- Responsive Testing

## 4. Assumptions
- Website is accessible and live
- No login required for browsing
- Testing is done on production environment
- No real payments will be completed

## 5. Risks
- Live data may change during testing
- Promotions may change dynamically
- No staging environment available
- Checkout flow may require real payment

## 6. Entry Criteria
- Website is accessible
- Test cases are prepared

## 7. Exit Criteria
- All test cases executed
- Bugs documented
- Test summary prepared
